Accessible Layouts

This is a fairly gigantic area to cover, but Sky is taking a responsive design approach which helps to meet many of the accessibility requirements.

Accessibility requirements

The main requirements are that:

  • The page can be zoomed / resized up to 200% without loosing content or functionality
    (Resizing)
  • There is a meaningful order of elements, both visually and when tabbing through
    (Meaningful reading order)
  • Navigation is consistent in placement and labelling
    (Consistent Navigation & Consistent labelling)
  • Skip links are available to get past repeated navigation
    (Skip links)

Additionally, it is useful to use ARIA landmarks to provide within-page navigation for screen readers.

Methods

The main ways to meet the criteria are:

  • Use responsive web design (percentage based layouts with media queries).
  • Create layouts in basic HTML first, and ensure that the linear order makes sense. Then style the page to match the intended design.
  • Check that the markup is appropriate
  • Put the navigation in the same place(s) as other pages.
  • Use one (perhaps two) skip links, and WAI-ARIA landmarks.

Testing

In supported desktop browsers check that:

  • At a small resolution (e.g. 1024 pixels wide) it is possible to zoom to 200% without horizontal scrolling or loss of content.
    NB: It is reasonable to use Internet Explorer and Firefox to test this aspect, until Safari and Chrome support zooming with media queries (see Resizing).
  • Using the tab-key to progress through links results in an understandable order.
  • Using the tab-key to progress, one of the first links on the page is a "Skip to content" link.

In supported mobile browsers check that zooming has not been disabled.
